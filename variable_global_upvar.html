Global and Local variables

Does the code work?


set alpha 1

proc myproc {} {
    puts $alpha
}

myproc

No!!!, Above code doesn't work because the two alphas are in different scopes.

The command set alpha 1 creates a variable in the global scope

command puts $alpha is executed in scope that is created when command myproc executes.

The two scopes are distinct. This means that when puts $alpha tries to look up the name alpha, it doesn't find any such variable.

global

The global command is used to create or access global variables. Global variables can be accessed from any part of the script.

 


# Example of using global command
set globalVar 10

proc myProcedure {} {
    global globalVar
    puts "Inside procedure: \$globalVar = $globalVar"
}

myProcedure
puts "Outside procedure: \$globalVar = $globalVar"

upvar

The upvar command is used to create a local variable that refers to a variable in an outer scope, allowing you to modify variables in calling procedures.

 


# Example of using upvar command
set outerVar 5

proc myProcedure {innerVar} {
    upvar $innerVar localVar
    puts "Inside procedure: \$localVar = $localVar"
    set localVar [expr $localVar + 1]
}

myProcedure outerVar
puts "Outside procedure: \$outerVar = $outerVar"
